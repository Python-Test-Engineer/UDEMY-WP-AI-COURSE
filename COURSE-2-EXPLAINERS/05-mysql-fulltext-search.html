üìñ What is Full-Text Search?

Full-Text Search is MySQL's built-in feature for searching text content efficiently. Unlike basic pattern matching with LIKE, it's optimized for searching through large amounts of text data.

‚ùå Regular LIKE Search
SELECT * FROM articles WHERE content LIKE '%database%';
  • Scans entire table (slow)
  • No relevance ranking
  • Can't search multiple words efficiently
  • Case-insensitive issues
‚úÖ Full-Text Search
SELECT * FROM articles WHERE MATCH(content) AGAINST('database');
  • Uses special index (fast)
  • Ranks results by relevance
  • Handles multiple words naturally
  • Built-in word stemming

üöÄ Getting Started

Step 1: Create a Table with FULLTEXT Index

-- Create a table for blog posts CREATE TABLE articles ( id INT PRIMARY KEY AUTO_INCREMENT, title VARCHAR(200), content TEXT, FULLTEXT(title, content) -- Create fulltext index ) ENGINE=InnoDB;

Step 2: Add Some Data

INSERT INTO articles (title, content) VALUES ('MySQL Tutorial', 'Learn MySQL database basics'), ('Python Guide', 'Python programming for beginners'), ('MySQL Performance', 'Optimize your MySQL queries');

Step 3: Search!

-- Basic search SELECT title, content FROM articles WHERE MATCH(title, content) AGAINST('MySQL'); -- Search with relevance score SELECT title, MATCH(title, content) AGAINST('MySQL') AS score FROM articles WHERE MATCH(title, content) AGAINST('MySQL') ORDER BY score DESC;

üéØ Search Modes

Natural Language (Default)

AGAINST('mysql tutorial')

Searches for words, ranks by relevance. Most common mode.

Boolean Mode

AGAINST('+mysql -oracle' IN BOOLEAN MODE)

Use operators: + (must have), - (must not have), * (wildcard)

Query Expansion

AGAINST('database' WITH QUERY EXPANSION)

Finds related terms automatically. Good for broad searches.

üí° Boolean Mode Operators

Operator Example Meaning
+ +mysql +tutorial Must contain both words
- mysql -oracle Must have mysql, must NOT have oracle
* data* Matches database, data, dataset, etc.
" " "mysql tutorial" Exact phrase match
( ) +mysql +(tutorial guide) Grouping terms
> >mysql database Increase importance of mysql
<< /strong> Decrease importance of mysql
üéØ Example Queries:
-- Must have "python", must not have "beginner" AGAINST('+python -beginner' IN BOOLEAN MODE) -- Find "mysql" or "postgresql" AGAINST('mysql postgresql' IN BOOLEAN MODE) -- Exact phrase AGAINST('"database performance"' IN BOOLEAN MODE) -- Wildcard for variations AGAINST('program*' IN BOOLEAN MODE)

‚ö° Performance Tips

üí° Tip 1: Full-text indexes work best on TEXT or VARCHAR columns with lots of content.
üí° Tip 2: MySQL ignores common words (stopwords) like "the", "is", "at". You can customize this list.
üí° Tip 3: Words must be at least 4 characters by default (configurable with ft_min_word_len).
üí° Tip 4: Use Boolean mode when you don't need relevance scoring (it's faster).

üéì Complete Example

-- Create table with fulltext index CREATE TABLE blog_posts ( id INT AUTO_INCREMENT PRIMARY KEY, title VARCHAR(255), content TEXT, author VARCHAR(100), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FULLTEXT idx_title_content (title, content) ); -- Insert sample data INSERT INTO blog_posts (title, content, author) VALUES ('Getting Started with MySQL', 'MySQL is a powerful open-source database...', 'John Doe'), ('Advanced MySQL Techniques', 'Learn query optimization and indexing...', 'Jane Smith'); -- Search with relevance score SELECT title, author, MATCH(title, content) AGAINST('MySQL database') AS relevance FROM blog_posts WHERE MATCH(title, content) AGAINST('MySQL database') ORDER BY relevance DESC LIMIT 10; -- Boolean search: must have MySQL, prefer optimization SELECT title, author FROM blog_posts WHERE MATCH(title, content) AGAINST('+MySQL >optimization' IN BOOLEAN MODE);

‚úÖ Pros and Cons

‚úÖ Advantages

  • Much faster than LIKE for text searches
  • Built-in relevance ranking
  • Handles complex queries easily
  • Natural language support
  • No external dependencies

‚ùå Limitations

  • Only works with InnoDB and MyISAM
  • Requires fulltext index (storage overhead)
  • Minimum word length requirement
  • Ignores stopwords
  • Not as powerful as Elasticsearch for complex needs

üîÑ Adding Index to Existing Table

-- Add fulltext index to existing table ALTER TABLE articles ADD FULLTEXT idx_content (content); -- Add index on multiple columns ALTER TABLE articles ADD FULLTEXT idx_title_content (title, content); -- Drop fulltext index ALTER TABLE articles DROP INDEX idx_content;
üéØ When to Use Full-Text Search:
  • Blog posts, articles, documentation
  • Product descriptions and reviews
  • FAQ and knowledge bases
  • Any text-heavy content where users need to search
üéØ When to Use Something Else:
  • Need advanced features ‚Üí Use Elasticsearch or Solr
  • Simple exact matches ‚Üí Use regular WHERE clauses
  • Small tables ‚Üí LIKE might be fine
