### Full-Text Search vs Vector Search Comparison

| Aspect | Full-Text Search | Vector Search |
|--------|------------------|---------------|
| **Algorithm** | MySQL MATCH AGAINST | Cosine Similarity |
| **Data Type** | Text (keywords) | 1536-dim vectors |
| **Accuracy** | Keyword matching | Semantic understanding |
| **Performance** | Fast (database) | Slower (API + calculation) |
| **Setup** | MySQL index only | OpenAI API key + embeddings |
| **Best For** | Exact terms, categories | Meaning, context, synonyms |

### Performance Characteristics

```
Full-Text Search Performance:
âœ“ Database-native indexing
âœ“ Sub-millisecond query time
âœ“ No external API calls
âœ“ Scales with database size

Vector Search Performance:
âœ— Requires embedding generation (slow)
âœ— External API dependency
âœ— Vector calculations per query
âœ— Better semantic understanding

Hybrid Approach Benefits:
âœ“ Use FTS for speed + Vector for accuracy
âœ“ Compare results to understand differences
âœ“ Choose appropriate method per use case
```
